# GPR_handoffのカーネル

## 1. 最初に伝える要点

今回の比較は、次の順で理解すると簡単です。

1. まずRBFやMatérnなど、通常の定常カーネルを同じ固定10-foldで比較した。
2. 通常カーネルの上位差は小さく、Matérn 3/2とRBFはほぼ同等だった。
3. 誤差を調べると、分子軸の向きだけでなくH3近傍Mg/O環境との組合せが重要だった。
4. そこで `軸 + 環境 + 軸×環境 + White` のProduct GPを作ると、固定10-foldとtrajectory Held-outの両方でRFを上回った。

したがって今回の主結果は「MatérnがRBFより明確に優れる」ことではなく、**分子軸と局所環境の相互作用をカーネルに入れることが有効だった**ことです。

## 2. 比較した通常カーネルの名前と数式

標準化後の2入力 (x,x') の距離を

\[
r(x,x')=
\sqrt{\sum_j\left(\frac{x_j-x'_j}{\ell_j}\right)^2}
\]

とします。等方カーネルでは全特徴に共通の (ell)、ARDでは特徴ごとの (ell_j) を使います。

### 2.1 Matérn 1/2（Exponential kernel）

\[
k_{M12}(x,x')=\sigma_f^2\exp(-r).
\]

急な変化を許す最も粗いMatérn候補です。

### 2.2 Matérn 3/2 kernel

\[
k_{M32}(x,x')=
\sigma_f^2(1+\sqrt3r)\exp(-\sqrt3r).
\]

粗さと滑らかさの中間にある候補です。

### 2.3 Matérn 5/2 kernel

\[
k_{M52}(x,x')=
\sigma_f^2\left(1+\sqrt5r+\frac{5r^2}{3}\right)\exp(-\sqrt5r).
\]

Matérn 3/2より滑らかな関数を仮定します。

### 2.4 RBF / Squared Exponential kernel

\[
k_{RBF}(x,x')=\sigma_f^2\exp\left(-\frac{r^2}{2}\right).
\]

非常に滑らかな関数を仮定します。RBF-ARDは同じ式で特徴ごとの (ell_j) を持ちます。

### 2.5 Rational Quadratic kernel

\[
k_{RQ}(x,x')=
\sigma_f^2\left(1+\frac{r^2}{2\alpha}\right)^{-\alpha}.
\]

異なる長さ尺度を持つRBFの混合として解釈できます。(alpha\to\infty) でRBFへ近づきます。

### 2.6 Linear / DotProduct kernel

\[
k_{Lin}(x,x')=
\sigma_f^2(\sigma_0^2+x^\top x').
\]

非線形カーネルが必要かを確認するための対照です。

### 2.7 White noise kernel

\[
k_W(x,x')=\sigma_n^2\mathbf 1[x=x'].
\]

観測ノイズ、または入力特徴だけでは説明できない独立変動を表します。上の各信号カーネルにこの項を加えます。例えばRBFモデル全体は

\[
k_{RBF+W}=k_{RBF}+k_W
\]

です。

## 3. 4個の軸特徴とProduct kernel

### 3.1 なぜ4個を取り出したか

元baseは `file_key` と `y` を除く111特徴です。このうちC3→H3とC6→H6の方向を表す7列は、2本がほぼ反平行なため強く重複しています。

| 4個の軸特徴 `a` | 残した物理情報 |
| --- | --- |
| `axis_azimuth_sin = sin(phi)` | 分子軸のxy方位と角度周期性 |
| `axis_azimuth_cos = cos(phi)` | 同上。sinと組にして方位を表現 |
| `axis_abs_elevation_rad_proxy` | 分子軸の面外傾きの絶対値proxy |
| `antiparallel_deviation_rad` | 2本のC–H方向の180°からのずれ |

この選択には次の理由があります。

- `sin/cos` は−180°と+180°を近い方向として扱える。
- ほぼ反平行な2本の方位を別々に入力する重複を減らせる。
- PowerPointと誤差解析の両方で、面内方位と `z≈0` が重要と示唆された。
- 完全な反平行からの歪みも失わずに残せる。

除いた7列は `C3H3_angle_xy`, `C6H6_angle_xy`, `angle_diff_C3_C6`, `sin_angle_diff_C3_C6`, `cos_angle_diff_C3_C6`, `dot_C3H3_C6H6`, `dot_xy_C3H3_C6H6` です。

残り104列から、現在の固定foldで分散ゼロだった `first_invd_LH3_atomOther`, `first_invd_LH6_atomOther` を除き、環境baseは102次元になります。さらにfold内 `X_proc` PCA8を加えます。

\[
a\in\mathbb R^4,\qquad
e\in\mathbb R^{102+8}=\mathbb R^{110}.
\]

列除外、標準化、PCAはすべて訓練foldだけでfitします。

### 3.2 Product GPの各成分

Matérn 3/2の単位分散部分を (M_{3/2}(r)=(1+\sqrt3r)e^{-\sqrt3r}) とします。

軸だけの類似性は

\[
k_a(a,a')=\sigma_a^2M_{3/2}(r_a;\ell_a),
\]

環境だけの類似性は

\[
k_e(e,e')=\sigma_e^2M_{3/2}(r_e;\ell_e),
\]

軸と環境の相互作用は

\[
k_{a\times e}((a,e),(a',e'))=
\sigma_{ae}^2
M_{3/2}(r_a;\ell_{a,ae})
M_{3/2}(r_e;\ell_{e,ae})
\]

です。モデル全体を

\[
\boxed{
k_{Product}=k_a+k_e+k_{a\times e}+k_W
}
\]

とします。

- (k_a): 環境によらず共通する軸効果
- (k_e): 軸によらず共通する環境効果
- (k_{a\times e}): 軸の向きによって環境効果が変わる非加法相互作用
- (k_W): White noise

実装では3個の信号分散、加法2個＋積項2個の計4個の長さ尺度、White noiseを推定します。すべて各訓練foldの対数周辺尤度を最大化する第二種最尤法で決めます。

## 4. なぜProduct kernelを考えたか

通常のMatérn 3/2 GPでは、50–70°帯に二乗誤差の59.4%が集中しました。この帯には高い `y` と低い `y` の枝が共存し、低い枝ではH3近傍のMg/Oが多く近く、面外傾きが小さい傾向がありました。

角度だけのカーネルでは、この枝分かれを表しにくくなります。加法カーネル (k_a+k_e) でも、軸効果と環境効果を独立に足すだけです。積項は「軸も環境も似ているときだけ強く似る」ため、今回観察された枝分かれを直接表せます。

各ブロックをMatérn 3/2にしたのは、通常カーネルscreenでR²・RMSEがわずかに最良で、RBFほど強い滑らかさもMatérn 1/2ほどの粗さも仮定しないためです。ただし上位通常カーネルの差は小さく、Matérn 3/2が真の物理関数だと確定したわけではありません。

## 5. RFとの比較結果

| Held-out設定 | Product GP R² / RMSE / MAE | 受領二段RF R² / RMSE / MAE |
| --- | ---: | ---: |
| 固定10-fold | **0.973807 / 1.638955 / 0.639434** | 報告値 0.908223 / 3.067897 / 1.261954 |
| trajectory group5 | **0.465545 / 7.403375 / 4.513588** | 0.025996 / 9.994348 / 5.451921 |
| trajectory group10 | **0.365228 / 8.068315 / 4.852925** | −0.212622 / 11.151589 / 5.720765 |

固定10-foldでは、加法GPは `R²=0.955026`、通常Matérn 3/2は0.933874でした。積項を追加したProduct GPが0.973807まで改善したため、単にGPRへ置き換えた効果だけでなく、軸×環境の構造が効いています。

一方、trajectoryを丸ごとHeld outするとProduct GPも大きく低下します。同じ分割のRFよりは良好ですが、未知trajectoryへの十分な外挿性能はまだありません。

受領RFの残差PLS5/RF補正は、固定10-foldではbase RFを改善しましたが、trajectory group5とgroup10ではどちらも悪化しました。残差補正が系列固有の細かなパターンを学習した可能性があります。

## 6. カーネルから何が言えるか

1. 通常カーネル上位4種の差は小さいため、滑らかさの選択だけでは結果を説明できない。
2. 加法GPよりProduct GPが良いため、軸と環境を独立に足すより、その組合せを表す方がデータに合う。
3. group10では積項分散が全foldで使われ、軸・環境の加法分散は多くのfoldで下限に達した。これは相互作用仮説と整合する。
4. ただしカーネル成分の大きさは因果的寄与率ではない。「Mg/Oが近いから必ず `y` が下がる」とは結論できない。
5. 固定foldからtrajectory Held-outへの大幅な低下は、現在の高性能が主に既知系列内補間であることを示す。

## 7. 予測平均面と観測点がずれて見える理由

3D面では方位と傾き、または方位とH3近傍強度だけを動かし、表示しない110入力を基準試料 `0-0-3-18-10` に固定しています。これは高次元関数の条件付き断面です。

他の観測点は、同じ2座標へ投影できても残り110入力が異なります。そのため、それらの観測値がこの面上に乗る必要はありません。従来図はこの非比較な全点を重ねたため、フィットがずれて見えました。更新後は条件付き面と同じ残り入力を持つ基準試料1点だけを表示します。

また、White noiseと正則化を持つGPの予測平均は、訓練点を必ず通る補間面ではありません。フィット性能は条件付き3D面ではなく、OOFの実測対予測図とR²・RMSEで判断します。

## 8. 解釈と次の検証

- 既知trajectory内の補間にはProduct GPを第一候補とする。
- 未知trajectory用モデルはまだ確定しない。
- `file_key` tokenの正式な物理意味を確認して外側groupを固定する。
- raw 3D座標からMg/Oの軸方向射影距離・垂直距離・H3/H6非対称性を作る。
- モデル、PCA/PLS次元、regime境界はnested group CVの内側だけで選ぶ。

詳細なfold別結果は `results/`、再計算手順は[実験詳細](EXPERIMENTS_JA.md)にあります。
